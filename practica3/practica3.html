Archivo Reset de CSS

Para poder tratar con las diferencias que existen entre navegador y navegador, comúnmente se incluye un archivo .css llamado Reset. Lo que este archivo nos permite es poder homologar los formatos y estilos de las tags de HTML, para conseguir una visualización estándar sin importar el navegador del que se accese a la página. Historia de la presentación en pantalla en la Web

En el pasado, cuando CSS no existía, se realizaba el formato de la página a través de tablas de HTML, que aunque funcionaban, hoy en día no serían un reemplazo adecuado para la gran cantidad de herramientas útiles que nos brinda el CSS y sus hojas de estilo.
CSS ha sustituido ya el empleo de tablas para organizar las páginas, por lo que los elementos de tabla de HTML van regresando, poco a poco, a su función original, que es la de mostrar datos estructurados, y abandonan su papel como armazón visual de las páginas.

Elemento div

El elemento de división de contenido HTML (<div>) es el contenedor genérico para el contenido de flujo. No tiene ningún efecto sobre el contenido o el diseño hasta que se haya diseñado con CSS.

Como contenedor "puro", el elemento <div> no representa inherentemente nada. En cambio, se usa para agrupar contenido para que se pueda diseñar fácilmente usando los atributos de clase o id, marcando una sección de un documento como escrita en un idioma diferente (usando el atributo lang), y así sucesivamente.

Un elemento <div> suele llevar un atributo ID (con lo que adquiere un “nombre” explícito dentro de la página) o bien pertenecer a una clase CSS, utilizándose ambos como medio para aplicar al contenedor uno o más estilos CSS.

Propiedad float

La propiedad CSS float coloca un elemento en el lado izquierdo o derecho de su contenedor, lo que permite que el texto y los elementos en línea lo envuelvan. El elemento se elimina del flujo normal de la página, aunque sigue siendo parte del flujo (en contraste con el posicionamiento absoluto).
La propiedad float se especifica como una sola palabra clave, elegida de la lista de valores a continuación.

  • left: El elemento debe flotar en el lado izquierdo de su bloque contenedor.
  • right: El elemento debe flotar en el lado derecho de su bloque contenedor.
  • none: El elemento no debe flotar.
  • inline-start: El elemento debe flotar en el lado inicial de su bloque contenedor. Ese es el lado izquierdo con scripts ltr, y el lado derecho con scripts rtl.
  • inline-end: El elemento debe flotar en el lado final de su bloque contenedor. Ese es el lado izquierdo con scripts ltr, y el lado derecho con scripts rtl.

Columnas con float

Es posible crear columnas de distintas anchuras y alturas usando la propiedad float.

Propiedad padding

La propiedad CSS de padding establece el área de padding en los cuatro lados de un elemento. Es una forma abreviada de padding-top, padding-right, padding-bottom y padding-left. El área de padding de un elemento es el espacio entre su contenido y su borde.
La propiedad de relleno se puede especificar utilizando uno, dos, tres o cuatro valores. Cada valor es una <length> o una <percentage>. Los valores negativos no son válidos. Esta página usa paddings para sus elementos de CSS.

¿Qué es un CSS?

Como se había definido desde la práctica 1, un CSS hace referencia a un archivo de hojas de estilo en cascada para poder describir la presentación visual de un documento HTML, para agregarle elementos estéticos y en algunos casos funcionales. El CSS está considerado como una de las 3 tecnologías fundamentales para el desarrollo web, siendo las otras HTML5 y Javascript.
Actualmente se usa la versión CSS3, que se empezó a desarrollar en 1999.

¿Para qué sirve un CSS?

Se puede utilizar en diferentes tipos de dispositivos, como pantallas e impresoras grandes o pequeñas. Es independiente del HTML y se puede usar con cualquier lenguaje de marcado basado en XML. Las especificaciones CSS son mantenidas principalmente por el Consorcio World Wide Web. El conjunto de reglas CSS consta de un selector y un bloque de declaración.

Mientras usa CSS, un usuario puede estar libre de estrés con sus versiones de idiomas anteriores, ya que es muy compatible incluso con las versiones anteriores. Debido a esto, incluso si las aplicaciones de CSS se desarrollan en alguna versión anterior del lenguaje de programación y si los desarrolladores integran lo mismo con nuevas mejoras, CSS se puede incorporar e implementar fácilmente con esos cambios fácilmente. Por lo tanto, no es necesario eliminar los cambios asociados con el predecesor de CSS y un desarrollador puede actualizar el código existente con éxito.

Desde el punto de vista de la accesibilidad, las aplicaciones de CSS proporcionan soluciones mucho mejores que permiten a los usuarios actualizar la interfaz de usuario para satisfacer los requisitos comerciales. También permite que las páginas web sean fácilmente representadas por diferentes dispositivos. Esto en realidad proporciona un impacto mucho más profundo al considerar la modificación del aspecto de una página web desde la perspectiva del usuario final y del negocio.

¿Cuál es la diferencia que sería mas difícil de realizar en un sitio web si se optara por no usar un CSS?

El no usar CSS en el desarrollo web debe de ser la peor decisión que puede tomar cualquier desarrollador, fundamentalmente hablando, sin CSS se haría extremadamente difícil (pero no imposible) el estilo y formato de dicho sitio web. En décadas anteriores, cuando CSS no existía, se realizaba el formato de la página a través de tablas de HTML, que aunque funcionaban, hoy en día no serían un reemplazo adecuado para la gran cantidad de herramientas útiles que nos brinda el CSS y sus hojas de estilo.
